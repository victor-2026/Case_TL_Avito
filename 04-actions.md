## Q3: Какие действия предпринять?

### Weeks 1–2: Research

| Действие | Владелец | Результат |
|----------|----------|-----------|
| Собрать метрики по всем 3 командам | Test Lead | Таблица current state |
| Интервью с dev команды 1 и QA команды 2 | Test Lead | Понимание процессов |
| Встреча с командой 3 (соседи) | Test Lead | Статус Соседского соглашения |
| Post-mortem топ-5 инцидентов команды 2 | Test Lead + Dev | Root causes + action items |

### Weeks 3–6: Quick Wins

| Действие | SMART цель | Срок |
|----------|-----------|------|
| Внедрить Zero Bug Policy для всего юнита | Каждый баг получает решение в день регистрации (чиним/очередь/never/задача), P0 — без решения > 24ч | Week 4 |
| Авто-тесты на top-3 частых бага | 3 теста в регрессии, pass rate > 95% | Week 5 |
| Соседское соглашение с командой 3 | Подписанный документ, 11 правил | Week 6 |
| Дашборд багов/инцидентов для юнита | Visibility для команды и менеджмента | Week 6 |

### Weeks 7–13: Foundation

| Действие | SMART цель | Срок |
|----------|-----------|------|
| API contract tests на интеграцию с командой 3 | Все critical endpoints покрыты | Week 10 |
| Расширение QA-команды (2-й QA) или cross-training dev | QA-активность в команде 1 + усиление команды 2 | Week 10 |
| Regression suite стабильность | 95%+ green, flaky rate < 5% | Week 12 |
| Defect leakage tracking | Процесс измерения: баги prod / всего багов | Week 12 |

### 90-Day Outcomes

| Метрика | Старт | Цель (день 90) |
|---------|-------|---------------|
| P0 bugs/мес | Собрать | −50% |
| MTTR P0 | Собрать | < 2 часов |
| SPT-обращений/мес | Собрать | −50% |
| Defect leakage | Не измеряется | < 5% |
| API coverage (critical) | Неизвестно | > 80% |
| Соседское соглашение | Нет | Заключено |
