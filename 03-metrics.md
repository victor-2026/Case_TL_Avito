### Phase 2 — Tracking (с недели 3)

Метрики, которые буду внедрять для контроля изменений:

| Метрика | Зачем | Цель (1 квартал) |
|---------|-------|-----------------|
| P0 bugs/мес | Критичные баги не должны расти | −50% |
| MTTR P0 | Скорость реакции на критические сбои | < 2 часов |
| Defect leakage | % багов, доехавших до prod | < 5% |
| API coverage (critical paths) | Ключевые интеграции под тестами | > 80% |
| Regression suite stability | Регрессия не должна ломаться без причины | > 95% green |
| SPT-обращений/мес | Снижение жалоб клиентов | −50% |

### TMM Level 2 — обязательный baseline

Team Maturity Model (TMM) — не справочная метрика, а обязательный стандарт. Каждая команда обязана достичь уровня 2 по секции «Обеспечение качества»:

- **Квартальная самооценка:** команда оценивает себя по критериям TMM
- **Подтверждение экспертами:** оценки утверждаются экспертами секции QA из Tech Platform
- **Аудит соответствия:** Test Lead контролирует, что baseline не снижается
- **Критерии уровня 2:** пирамида тестирования, coverage, CI/CD gates, работа с багами, SPT/ZBP процессы

Операционные метрики для Tech Lead'ов:
- Lead-time, Velocity, Scope-drop, Defect rate — для оценки эффективности поставки
