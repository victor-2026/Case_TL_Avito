# Принципы разработки Avito

**Источник:** [avito-tech/playbook/development-principles.md](https://github.com/avito-tech/playbook/blob/master/development-principles.md)

## Основные принципы

1. **Мы отвечаем за работоспособность своей системы** — You build it, you own it. QA не контролёр, а часть команды.
2. **Ваша система — ответственность надолго** — Поддерживаемость = инвестиция.
3. **Начинаем с проектирования, учитывая архитектуру** — TDR для тестов тоже.
4. **Избегаем ненужной сложности** — Не строй тестовый комбайн.
5. **Используем готовые инструменты** — Не пиши свой фреймворк.
6. **Относимся к чужому коду с уважением** — Code Review тестов как production code.
7. **Если стало ненужным — удаляем** — Не храни мёртвые тесты.

## Принципы тестирования

- **Любые изменения должны легко тестироваться автоматически** — Testability by design
- **Что не протестировано — то не работает. Что не работает — не катим** — Quality gate
- **Zero Bug Policy** — Не копим баги. По каждому решение сразу: чиним сейчас / в порядке очереди / никогда

## Принципы эксплуатации

- **Всегда в курсе, что происходит с системами** — Дашборды + алерты
- **Fault tolerance** — Сервис работает, даже если зависимость упала
- **Graceful degradation** — Если нет данных — отдаём заглушку

## Работа с данными

- У данных только один источник правды
- Обратная совместимость миграций
- Не пишем бизнес-логику в базе
- Используем асинхронное взаимодействие по возможности
