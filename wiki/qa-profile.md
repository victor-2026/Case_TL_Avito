# QA-инженеры — профиль и матрица компетенций

**Источник:** [avito-tech/playbook/qa-profile.md](https://github.com/avito-tech/playbook/blob/master/qa-profile.md)

## Грейды

| Уровень | Название | Фокус |
|---------|----------|-------|
| E1 | QA-инженер (стажёр) | Выполняет задачи под руководством |
| E2 | QA-инженер (junior) | Может самостоятельно тестировать простые фичи |
| E3 | QA-инженер | Самостоятельно тестирует фичи, пишет авто-тесты |
| E4 | QA-инженер (senior) | Разрабатывает стратегию тестирования для фичи |
| E5 | QA-инженер (lead) | Разрабатывает стратегию тестирования для кластера |
| E6 | QA-инженер (head) | Формирует QA-стратегию в рамках Авито |

## Ключевые ожидания по грейдам

### E1–E2
- Ручное тестирование, тест-дизайн
- Основы автоматизации
- Баг-трекинг

### E3
- Полный цикл тестирования фичи
- UI/API авто-тесты
- Интеграция в CI/CD
- Cross-team коммуникация

### E4
- Стратегия тестирования фичи
- Performance / Security / Reliability
- Оптимизация процессов
- Code review тестов

### E5 (Lead)
- Стратегия тестирования кластера
- Метрики качества: релизные, продуктовые, процесса
- Управление рисками и долгами качества
- Развитие инженеров
- E2E тестирование кластера

### E6 (Head)
- QA-стратегия Avito
- Выбор инструментов и фреймворков
- Управление QA-наймом
- Построение и развитие QA-сообщества
- Бенчмаркинг индустрии

## Применение в кейсе

- QA команды 2 — предположительно E3 (выполняет, не разрабатывает стратегию)
- Позиция кандидата — Test Lead = E5 (стратегия для кластера/юнита)
- У команды 1 нет QA — стратегия: или перераспределить QA, или обучить dev QA-практикам
