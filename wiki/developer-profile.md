# Инженеры — профиль и матрица компетенций

**Источник:** [avito-tech/playbook/developer-profile.md](https://github.com/avito-tech/playbook/blob/master/developer-profile.md)

## Basic / Core

### E1
- Умеет декомпозировать задачу
- Может написать код по готовому плану
- Может написать тесты на свой код
- Знаком с Git и CI/CD

### E2
- Умеет декомпозировать нетривиальную задачу
- Пишет тесты на свой код и ревьюит тесты смежников
- Может настроить окружение для задачи
- Понимает принципы fault tolerance

### E3
- Умеет декомпозировать задачу на квартал
- Может настроить CI/CD для сервиса
- Пишет код с учётом совместимости API
- Рефакторит чужой код

## Networking (выборочно)

### E1
- Понимает HTTP и принципы REST
- Понимает принципы io, blocking/non-blocking

### E2
- Может спроектировать схему БД для микросервиса
- Разбирается в инфраструктурных компонентах (балансировщик, service discovery)

### E3
- Может спроектировать публичный API сервиса
- Применяет Circuit Breaker и Rate Limiter для защиты ресурсов

## Применение к кейсу

- Dev команды 1 (без QA) — вероятно E3+ (пишут тесты, CI/CD, fault tolerance)
- Dev команды 2 — разный уровень, раз багов много и QA не справляется
- Стратегия: поднять QA-культуру dev команды 2 через code review тестов, парное тестирование
