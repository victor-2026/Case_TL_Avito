# Процессы и стандарты в команде разработки

**Источник:** [avito-tech/playbook/processes-and-standards.md](https://github.com/avito-tech/playbook/blob/master/processes-and-standards.md)

## Стандарты

- **Код-стайл** — единый стиль в рамках языка/repo через линтеры и formatter'ы
- **Commit Convention** — явный формат коммитов для генерации changelog
- **API First** — спецификация API до реализации
- **Documentation as Code** — документация в репозитории

## Code Review

- Code Review обязателен для всех изменений
- Минимум один аппрув перед мержем
- Ревью тестов — часть Code Review
- COP (Code Ownership Plugin) уведомляет владельца

## Сборка и CI

- Каждый коммит проходит сборку
- Тесты — gate для мержа
- Зелёный билд — норма

## Деплой

- Непрерывный деплой (Continuous Delivery)
- Feature flags для контроля раскатки
- Rollback — одна кнопка
- Canary деплой для критичных изменений

## Инциденты

- Post-mortem обязателен для всех инцидентов
- Без blaiming (без поиска виноватого)
- Action items с владельцем и дедлайном

## Применение к кейсу

- Post-mortem как инструмент для багов команды 2: разобрать 5 худших инцидентов
- API First — применимо к интеграции команды 2 с платформой команды 3: контракты API до реализации
- COP — использовать для соседского соглашения с командой 3
