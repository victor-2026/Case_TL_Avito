# Структура команд Avito

**Источник:** [avito-tech/playbook/structure.md](https://github.com/avito-tech/playbook/blob/master/structure.md)

В Авито горизонтальная структура. Вся разработка разбита на множество кросс-функциональных команд (юнитов), в каждой из которых есть представители разных функций — инженеры, QA, продуктовые менеджеры, аналитики, дизайнеры. Руководит разработкой CTO и TMG (Tech Managers Group).

## Кластеры

Кластеры — это объединения юнитов, которые решают похожие задачи. Делятся на:

- **Платформенные** — разработка платформы, на базе которой строится продуктовая функциональность (например, Архитектура и Инфраструктура)
- **Продуктовые** — непосредственно продуктовые фичи (например, Buyer Experience, Seller Experience)

## Юниты

Состав команды определяется направлением деятельности. Чаще всего формируются вокруг потребности пользователя или части сервиса.

**Пример состава юнита (Messenger):**

```
Unit-Leader
  ├── Server-Side Developer
  ├── Client-Side Developer
  ├── iOS Developer
  ├── Android Developer
  ├── Designer
  └── QA
```

## Роль Unit-Leader

Unit-Leader отвечает за:

- **Стратегия и тактика** — понимает стратегию компании, формирует OKR, подбирает людей, управляет ресурсами
- **Коммуникации** — внутри команды и с другими юнитами
- **Работа с командой** — тренер и наставник, а не диктатор

---

**Примечание для кейса:** Test Lead — аналог Unit-Leader по качеству. Разница: Unit-Leader управляет командой (dev+QA+PM), Test Lead управляет качеством на уровне нескольких команд/кластера.
