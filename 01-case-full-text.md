## Полный текст кейса (цитата)

> В твоём бизнес-юните есть две команды. Одна команда владеет небольшим сервисом back-end, является агрегатором и предоставляет данные для других сервисов для команды из этого же юнита. В команде нет QA инженера. Но багов в бэклоге либо очень мало либо нет совсем.
>
> Вторая команда имеет блок на странице объявления, экраны на мобильных приложениях и некоторый бэк, которые отдаёт данные на эти UI. В ней есть один QA, в бэклоге очень много багов, а также есть регулярные жалобы от саппорта и клиентов на функционал этой команды.
>
> Третья команда (соседний юнит) предоставляет платформу, на которой базируется функционал второй команды. Третья команда в целом отвечает за карточку объявлений, в то время как вторая команда встраивается в неё. Ты не знаешь как обстоят дела с качеством этой команды.
>
> Твоя задача как test-lead — улучшить качество в твоих подопечных командах.
> С чего начнёшь свои исследования?
> Какие метрики будешь смотреть?
> Какие действия предпринять для улучшения ситуации?

## Структура и роли

| Команды                       | Функции                                                             | Состав QA           | Проблема                                                                     |
| ----------------------------- | ------------------------------------------------------------------- | ------------------- | ---------------------------------------------------------------------------- |
| **Команда 1**                 | Back-end агрегатор (данные для команды 2)                           | Нет QA              | Багов почти нет — гипотеза: либо покрытие идеальное, либо никто не ищет баги |
| **Команда 2**                 | (Рекламный?) блок для Карточки объявлений (web + mobile + свой бэк) | 1 QA                | Много багов, жалобы саппорта и клиентов                                      |
| **Команда 3** (соседний юнит) | Платформа "Карточка объявлений"                                     | Качество неизвестно | Интеграция с командой 2, но данных о качестве нет                            |

## Ключевые ограничения

- Test Lead отвечает **только за команды 1 и 2** (свой бизнес-юнит)
- Задача - улучшить качество команд 1 и 2
- Команда 3 — соседний юнит, повлиять можно только через **Соседское соглашение**
